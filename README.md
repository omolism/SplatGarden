# SplatGarden

A 3D Gaussian Splatting web viewer for an asset-pipeline showcase. A Houdini and Unreal Engine 5 garden, captured at a multi-camera rig, reconstructed with COLMAP, trained in parallel by Postshot and Lichtfeld Studio, optimised with Houdini GSOP, and rendered in the browser via Spark on Three.js + WebGL 2.

**Live**: https://splatgarden.com (apex domain; the previous `omolism.github.io/SplatGarden/` URL now redirects)

## Pipeline

Three stages match the canonical intro figure on the web:

```
01 Asset making     PCG · AI texture stylization · VAT bake · Houdini SIM
02 Scene assembly   Unreal Engine 5 · MediaPipe hand-tracking
03 3DGS capture     Multi-cam rig (990 poses) · COLMAP · Postshot ‖ Lichtfeld
                    → SplatGarden_PC.splat     (3M Gaussians, ~92 MB)
                    → SplatGarden_Mobile.spz   (same data, ~45 MB, phones)
```

## Run

```bash
npm install
npm run dev          # http://127.0.0.1:5173
npm run build && npm run preview
```

## Controls

| Key | Surface |
|---|---|
| `T` | Tech Breakdown drawer |
| `H` | Quick Guide |

| `1 / 2 / 3` | Front / Center / Zoom viewpoints |
| `W A S D + Q E` | Flythrough (`Shift` = 3×) |

### First-person walk

Toggle `First-person walk` in **Camera Movement** to enter Unreal-style free-fly.

- Desktop: `WASD` or arrow keys to walk, `Shift` to sprint, `Space` / `Ctrl` for vertical, mouse to look, `Esc` to release pointer lock.
- Phone: the canvas splits L / R — drag the left half to walk (a virtual joystick appears at your finger), drag the right half to look. Both fingers work at once.
- A soft 40 m boundary keeps you in the scene; uncheck the toggle to return to orbit.

### Right-rail studio (lil-gui)

3DGS / USD layer switcher · FX (eight shader-driven presets, optional **Color tint** per preset) · **Performance** preset (Battery / Balanced / Max Quality) · Customize (Post-FX · HDR Sky · Particles · Hand tracking · First-person walk) · Cinematic FX · Tech Spec · Camera Movement.

Title-row pills: **Replay** (re-fire the intro), **Snap** (PNG snapshot of the current view), **Reset** (settings → defaults).

### URL deep-links

| Hash | Effect |
|---|---|
| `#v=px,py,pz,tx,ty,tz` | Fly the camera to the encoded pose. Generated by the **Share viewpoint** button. |
| `#asset=Foliage` | Open the Tech Breakdown drawer and expand the named asset card. Case-insensitive. |
| `?renderer=webgpu` | Future-proofing flag — recorded on `window.__rendererPref`. Falls back to WebGL 2 until Spark exposes a WebGPU backend. |

Mix freely: `#v=...&asset=Foliage` lands on the pose then expands the drawer.

### Phone-specific UI

Floating bottom-bar (`Tour · Effects · Studio · Info · Share`), `Battery` default profile (DPR 1.0, no bloom, no particles), bottom-sheet variant of every asset card, virtual joystick when First-person walk is on.

## Mobile bandwidth

`pickSplatUrl()` in `main.js` HEAD-probes `public/SplatGarden_Mobile.spz` on phones and falls back to `.splat` if the SPZ is unshipped. Regenerate via:

```bash
node scripts/generate-spz-v3.mjs
```

The script uses Spark's own `transcodeSpz`, so the output version always matches the runtime (`0.1.10` → SPZ v3, `2.x` → v4). External tools work too if pinned to the matching version. See `_archive/README.md` if a SPZ stops loading after a Spark upgrade.

## Offline / install

`public/sw.js` registers as a service worker on first visit and runs **stale-while-revalidate** for the heavy assets (splat / SPZ / WebP / HDR / Vite-hashed JS+CSS) plus **network-first with a 3 s timeout** for HTML. The second visit boots from cache so the 45 MB SPZ never comes over the wire twice. `public/manifest.webmanifest` plus the `<link rel="manifest">` + `theme-color` tags in `index.html` enable "Add to Home Screen" on phone and Chrome / Edge install prompts on desktop.

## Custom splat upload

The 3DGS / USD panel's `⤓ Use My Own` button opens a drop zone for `.splat / .ply / .spz / .ksplat`. The new mesh is added as a secondary layer, the primary SplatGarden is auto-hidden (re-show via the Scene panel), and the camera reframes onto the new bounding box.

The companion `Max range (m)` input caps the auto-reframe radius — useful when an export has outlier points that would otherwise push the camera into the void. Empty = auto-detected; numeric = `min(auto, userMax)`. Value persists in localStorage.

## Deploy

GitHub Pages via `.github/workflows/deploy-pages.yml`. The workflow sets `VITE_BASE=/` because the site is served from the apex custom domain `splatgarden.com` (rather than the `omolism.github.io/SplatGarden/` project-page subpath). `public/CNAME` carries the domain into the build so GitHub Pages maps the deploy to it; the DNS records on Spaceship point the apex at GitHub's anycast IPs (`185.199.108.153`, `109.153`, `110.153`, `111.153`) plus a `www` CNAME to `omolism.github.io`. To fall back to the project page, flip `VITE_BASE` back to `/SplatGarden/` and remove `public/CNAME`. No Git LFS.

## Debug helpers

`window.__splatgardenState` returns a fresh snapshot of camera pose, target, active FX preset, layer visibility, and performance pref — useful for triage and analytics. `window.__captureSnapshot()` triggers the same PNG download as the title-row Snap pill. `window.__toast(msg)` shows a 1.5 s status pill at bottom-right.
